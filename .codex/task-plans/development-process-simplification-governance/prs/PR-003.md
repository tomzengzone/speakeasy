# PR-003: 前瞻式开发治理切换与普通 CI

## Objective

完成 mandatory FR、typed TC、derived traceability 和 legacy authority cutover，同时将交付治理收敛为常规 GitHub PR CI。删除可信 controller、controller/candidate 双 checkout、单一直接子提交限制、重复 HEAD/治理图摘要校验、机器可读 attestation，以及 candidate commit、远端 CI、baseline activation 分阶段审批。

## Included Scope

- 保留 canonical FR/TC/traceability、Governance Contract authority separation 和 legacy Spec/AC retirement。
- 更新 ADR 0007、workflow、Definition of Done、governance policy、相关 Skill、validator 和测试。
- CI 在 push/PR 上自动 checkout，并执行治理静态校验、Flutter analyze/test/build。
- 删除 exact-commit verifier 及其专用测试。

## Excluded Scope

- 不修改旧 Product Base / Increment 历史内容。
- 不创建或更新远程 PR，不执行 commit、push、merge、force push、分支保护配置或生产发布。
- 不降低产品事实、Contract、typed TC、traceability 或适用独立审查要求。

## Acceptance Criteria

- task plan 和 PR-003 不再要求高强度 exact-commit/attestation/activation 治理。
- workflow 不包含 `workflow_dispatch`、candidate/base 输入、手工 ref fetch、重复 checkpoint、attestation 生成或上传。
- CI 继续包含 checkout、治理静态校验、Flutter analyze、test 和 build，任一步失败即 job 失败。
- 专用 exact-commit verifier 与测试被删除，其他 validator 不再依赖其摘要或激活语义。
- ADR、workflow、DoD、policy 和 Skill 统一描述普通 PR CI。
- 针对性测试、Skill validator 和 task-plan validator 通过。
