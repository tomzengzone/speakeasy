# PR-003: PR3 精简开发治理原子切换

## Objective

在一个可回滚的候选 commit 中启用精简后的正式开发流程：删除强制 Spec/AC 阶段与重复 traceability 翻译，建立产品事实、Contract、TC oracle、Issue/PR 和 CI 状态的清晰 ownership，同时启用 approved VS / optional stable FR / canonical TC 路由；所有治理 authority 对同一 commit 原子切换，杜绝部分切换和双写。

## Included Scope

- 更新 workflow、Definition of Done、Artifact/Gate/Actor/Exchange/Intent、相关 Skills、validator、tests 和 CI required checks。
- 将默认交付路径精简为：选择 approved VS -> 识别可选 FR/受影响 Contract -> 先写或选择 TC -> 最小实现 -> 本地快速验证 -> exact-commit CI；风险项才追加专门审查、迁移和发布控制。
- 为 Agent/Skill/Issue 提供最小上下文路由，禁止把完整 Stage/Increment/Spec/AC 链作为默认 coding 输入。
- 建立 canonical `FUNCTIONAL_REQUIREMENT_CATALOG` 与 `TEST_CASE_CATALOG` 路由；approved User Story/VS 继续由产品事实 owner 批准。
- TC 以 approved VS 为直接产品上游，引用可选 FR/Contract，持有自包含 oracle 和稳定 selector；CI 持有 exact-commit 执行事实。
- 退役 `PRODUCT_BASE_SPEC`、`PRODUCT_BASE_ACCEPTANCE`、`PRODUCT_BASE_TRACEABILITY`、`INCREMENT_SPEC`、`INCREMENT_ACCEPTANCE`、`INCREMENT_TEST_CASES`、`INCREMENT_TRACEABILITY` 的 active authority；Increment/Stage 仅保留规划标签语义。
- 将已迁移 legacy 文档和 manifest 按 sunset 条件移入只读 archive，提供 cutover validator 证明无 active legacy route、无新下游引用和无事实丢失。
- 退役或重写强制 Spec/AC Skills，并更新 FR、TC、Issue、traceability 和相关治理执行方法。
- 以单候选 commit 执行全量 validator、tests、checker 和 CI；只有全部通过才把 governance status 激活并使 ADR 0007 实际 supersede ADR 0001。

## Excluded Scope

- 不实施任何真实产品 Slice；真实试点属于 PR-004。
- 不修改 ADR 0001 历史文本，不把历史文档删除到不可审计，不允许 legacy 和新 authority 同时支持新行为。
- 不因低风险判断免除事实 Contract 更新，也不把每次普通变更强制升级为 ADR/发布治理。
- 不提交、推送、创建或合并远程 PR，除非用户另行授权并修订 delivery target。

## Acceptance Criteria

- PR-002 的所有保留事实已落入 canonical VS/FR/TC/Contract，所有 legacy 记录满足终态和 destination 一致性；无未解决的当前行为仅存在于 legacy 文档。
- Active registry 只有一套交付 authority；`G-SPEC`、`G-AC-TC` 和强制 Spec/AC Artifact/Skill 已退役，新的 VS/FR/TC Gate、owner、exchange、intent 与 validator 一致。
- Workflow/DoD 不再包含为形式完整而生成 Spec/AC 的强制等待点；普通可逆变更具有最短反馈路径，高风险变更仍按风险触发对应控制。
- Agent/Skill 获得的 coding context 只包含 selected VS、适用 FR/Contract、相邻代码/测试和命令；流程 validator 拒绝无关 Increment/Stage 或 archived Spec/AC 回流。
- `docs/process/definition_of_done.md` 与总计划 Target Definition of Done 的十项语义逐项一致，并由 validator/tests 覆盖强制 Spec/AC 为零、TC oracle、Contract 事实同步、test-first、exact-commit CI、风险分级、无 legacy authority 和独立验收。
- Stage/Increment/Work Package 不再是产品行为、TC oracle 或 Contract 的权威依赖；Issue/PR 只记录交付选择关系。
- Canonical TC 不含执行状态，直接引用 approved VS 并具有自包含 oracle、层级和 selector；CI 结果绑定 exact commit SHA。
- Contract obligation 按 API/Domain/Persistence/AI/UX 事实变化触发；ADR、独立审查、迁移、flag/canary/rollback/release Gate 按风险另行触发。
- Legacy Spec/AC/Increment TC/traceability authority 只读归档，active Issue/TC/Contract/代码变更无法将其作为 authoritative input；manifest 满足 sunset 后移出 active route。
- workflow、DoD、Artifact/Gate/Actor/Exchange/Intent、Skills、validators、tests 和 CI 对同一候选 commit 全部通过；任一失败时 governance status 不激活。
- 独立治理/架构 checker PASS，用户显式验收后才完成 PR-003。
