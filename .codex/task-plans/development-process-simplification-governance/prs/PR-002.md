# PR-002: PR2 精简流程影子验证与事实迁移演练

## Objective

在完全非 canonical 的 shadow 区域验证精简后的端到端开发流程，而不只验证 Story/VS Schema：量化减少的强制文档跳数、重复事实和 AI 上下文，证明 VS/可选 FR/TC/Contract 能保留原有行为精度，并完成 262 个 legacy ID 的逐条迁移演练，为 PR3 无事实丢失的原子切换提供证据。

## Included Scope

- 新建带显式 `shadow-only` / `canonical: false` 标记的 FR、TC、VS coverage Schema、fixture 和映射数据。
- 建立旧流程与目标流程的可复现对照 fixture，统计强制 Artifact/Gate 跳数、重复行为定义、最小 AI coding 上下文文件/引用集合和反馈链节点。
- 定义 AI coding context bundle：selected approved VS、适用 FR、受影响 Contract、相邻代码/测试和验证命令；Stage/Increment/Spec/AC 不得作为默认输入。
- FR 只承载跨 Slice 稳定规则；TC 直接引用 approved VS、可选 FR 和受影响 Contract，并包含自足 Given/When/Then、边界、层级和可执行 selector。
- 为有效/无效 fixture 建立 validator 和测试，验证 approved VS coverage、TC oracle 完整性、typed reference、禁止 Spec/AC/Increment authority 回流及禁止把 CI 执行状态写入 TC catalog。
- 逐条核验 PR-001 manifest 的 262 个 legacy ID：有效事实映射到一个或多个 shadow destination；obsolete/rejected 有明确理由；`grandfathered-unverified` 若仍可能影响当前行为则阻塞 PR-003。
- 必要时只在 PR1 迁移合同路径内修正 Schema/validator/manifest，以表达经批准的迁移去向。

## Excluded Scope

- 不修改 `docs/process/workflow.md`、Definition of Done、active governance index/artifact/gate/actor/exchange/intent、现有 Skills 或 CI required checks。
- 不把 shadow FR/TC/coverage 注册为 canonical Artifact，不作为 Issue、代码、Contract 或测试实现的正式输入。
- 不停用 `G-SPEC`、`G-AC-TC`，不归档 legacy authority，不实施产品功能或应用测试。
- 不提交、推送或创建远程 PR。

## Acceptance Criteria

- 所有 shadow Artifact 均机器可判定为 non-canonical，且不出现在 active `artifact_routes`、Gate inputs、workflow、DoD、Skill 或正式 Issue 输入中。
- FR/TC Schema 和 fixtures 能表达 ADR 0007 的 VS-first、optional FR、self-contained TC oracle、affected Contract 和 exact selector 边界。
- Shadow 对照证明目标流程不再强制创建 Spec/AC，单一事实只由一个 owning source 定义，且普通单 Slice 变更的必经交接/Gate 数量少于 legacy 链路。
- AI context bundle validator 能拒绝无关 Stage/Increment/Spec/AC 全文链、重复事实副本和缺失 owning VS/Contract/测试上下文；输出可供 AI coding 直接消费的最小引用集合。
- 同一代表性 fixture 上，单 Slice/跨 Slice 目标链路分别不超过 2/3 个事实与验证节点，较 legacy 五节点链路减少至少 60%/40%；默认 AI context bundle 文件/引用数或 UTF-8 字节数不高于 legacy 必需上下文的 60%，行为与边界覆盖率保持 100%。
- Validator 对缺少 approved VS、依赖 Spec/AC/Increment、空 oracle、错误 Contract kind、TC execution status、重复/悬空 ID 等负例 fail closed。
- 262 个 manifest 记录全部经 owner 核验；`migrated` 记录至少一个 typed destination，终止记录无 destination，仍有效的行为/规则/Contract/oracle 不得仅以 `grandfathered-unverified` 逃逸。
- Shadow coverage 能证明每个计划保留的行为都有 approved VS 与 regression TC，跨 Slice 规则有稳定 FR，边界变化有对应 Contract。
- Legacy workflow、DoD、`G-SPEC`、`G-AC-TC` 和 Spec/AC Skills 在整个 PR-002 中保持唯一 authority。
- 定向 tests、现有治理回归和独立只读 checker 通过；用户显式验收后才可完成 PR-002。
