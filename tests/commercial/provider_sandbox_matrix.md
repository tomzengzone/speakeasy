# Provider Sandbox Matrix：商业化订阅上线准备

## 状态
External pending - 本矩阵定义 TC-COM-019 必需证据，不声明真实 Apple sandbox 或 Google Play internal test 已通过。

## 适用范围
- Increment：`commercial-subscription-readiness`
- Stage Scope：COM-SI-011
- Requirement：FR-COM-011
- Acceptance：AC-COM-013
- Test Case：TC-COM-019

## Evidence Contract
| Field | Requirement |
| --- | --- |
| Apple evidence ref | `APPLE_SANDBOX_EVIDENCE_REF` must point to purchase, restore, refund, expiry, grace-period, and account-switch evidence. |
| Google evidence ref | `GOOGLE_PLAY_INTERNAL_EVIDENCE_REF` must point to purchase, restore, refund, expiry, grace-period, and account-switch evidence. |
| Evidence format | Screenshots, provider console logs, backend verification logs, webhook logs, and account identifiers must be stored outside the repo and referenced from release vars. |
| Manual execution checklist | `tests/commercial/manual_external_evidence_checklist.md` defines step-by-step execution, expected results, actual result fields, and independent review requirements. |
| Local gate | `python3 scripts/check_provider_sandbox_evidence.py` validates this matrix and reports missing external evidence refs. |
| Strict release gate | `python3 scripts/check_provider_sandbox_evidence.py --strict-external` fails until external evidence refs are supplied. |

## External Blockers
- Apple Developer/App Store Connect sandbox tester access is required.
- App Store Server API / sandbox transaction inspection is required.
- Google Play Console internal testing track and license tester access are required.
- Refund, expiry, grace-period, and account-switch scenarios require provider-console state changes that cannot be generated by local deterministic tests.
