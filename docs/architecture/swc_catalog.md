## SWC ID Rules
- Prefix：`FE` 表示 frontend，`BE` 表示 backend，`DB` 表示 database/migration ownership，`AI` 表示 AI runtime/provider，`OPS` 表示 operations。
- 一个 SWC 可以映射到 package、feature module、service family、controller/service/repository group 或 operational boundary；不要求一对一映射到 class。
- 新的 implementation-impacting increment 必须引用这些 ID，或提出新 ID 并给出 reuse 与 duplicate-build justification。

## Required Catalog Fields
每个 accepted SWC entry 必须包含：
`SWC ID`, `Layer`, `Owning agent`, `Code path`, `Responsibilities`, `Explicit non-responsibilities`, `Provided interfaces`, `Required interfaces`, `Owned data / DB tables / migrations`, `Called APIs or provider boundaries`, `Test ownership`, `Required reuse / forbidden bypasses`, and `Status`.

## Global Forbidden Bypasses
- Flutter 不得生成 accepted `media://audio/...` ref、final entitlement facts、final mastery facts、provider secrets 或 provider-accessible signed URL。
- 当 `BE-AI-GATEWAY`、`BE-MEDIA-STORAGE`、`BE-USAGE-CONTROL` 或 `BE-COMMERCE-ENTITLEMENT` 边界适用时，backend business service 不得直接调用 raw provider。
- 没有明确 migration decision 时，新实现不得创建重复的 auth、audit、media、AI provider、entitlement、usage、training、goal 或 learning store。
- SWC allocation 不得复制 Domain Schema entity semantics 或 OpenAPI request/response schema。
