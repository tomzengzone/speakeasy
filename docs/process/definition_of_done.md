# Definition of Done

A feature is done only when:

- [ ] Feature spec exists.
- [ ] Acceptance criteria exist and are mapped to tests.
- [ ] Domain schema is updated if data changed.
- [ ] API contract is updated if frontend/backend boundary changed.
- [ ] Screen spec is updated if UI changed.
- [ ] AI output schema is updated if LLM behavior changed.
- [ ] Implementation is complete.
- [ ] Unit tests are added or updated.
- [ ] Integration/widget/e2e tests are added where risk requires them.
- [ ] Regression checks pass or gaps are documented.
- [ ] UX review is complete for user-facing changes.
- [ ] Implementation report is updated.
- [ ] Release checklist impact is reviewed.

## Completion Rule
Do not mark a feature complete if any required checkbox is unchecked without an explicit documented exception.
